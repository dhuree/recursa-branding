Quick Start

Get up and running in under 5 minutes.

1. Clone the repository

git clone https://github.com/dhuree/recursa.git
cd recursa

2. Open with your AI assistant

Recursa works with any AI assistant that can read files: Claude Code, Cursor, Windsurf, Cline, or others.

3. Bootstrap your project

Tell your AI assistant:

Bootstrap a self-improving blog system at ~/my-blog

The assistant will ask you questions about your domain, then generate customized templates.

4. Start iterating

Navigate to your new project and follow the LOOP.md file for each iteration.

Core Concepts

Self-Improving Systems

Traditional systems follow a simple pattern:

Input → Process → Output

Self-improving systems add feedback loops:

Input → Process → Output → Evaluate → Learn → Improve ↩

Recursa provides the architecture to implement these loops in any domain.

The Seven Layers

Recursa systems are built on seven architectural layers:

  1. Foundation — Core reasoning (human, AI, or hybrid)
  2. Identity — Values, principles, behavioral guidelines
  3. Memory — Tiered knowledge storage
  4. Tools — Extensible capabilities
  5. Reflection — Self-evaluation and feedback
  6. Orchestration — Multi-component coordination
  7. Safety — Guardrails, audit, rollback

The Loop

Every Recursa system has an iteration cycle defined in LOOP.md. The basic structure:

  1. Prepare — Load context, check memory
  2. Execute — Do the work
  3. Evaluate — Assess quality against metrics
  4. Capture — Document learnings
  5. Improve — Update process for next iteration

Nested Loops

Self-improvement happens at multiple timescales:

  • Task loop (minutes) — Execute, evaluate, retry
  • Session loop (hours) — Complete work, summarize
  • Skill loop (days) — Recognize patterns, abstract
  • Identity loop (weeks) — Analyze trends, evolve

Core Files

Identity Layer

  • SOUL.md — Core identity, values, philosophy
  • CONSTITUTION.md — Inviolable rules
  • IDENTITY.md — Name, personality, presentation

Operational Layer

  • LOOP.md — The iteration cycle
  • METRICS.md — Success measurements
  • LEARNING.md — Accumulated knowledge
  • GOALS.md — Current objectives

Safety Layer

  • GUARDRAILS.md — Safety infrastructure
  • action_log.jsonl — Audit trail

Bootstrapping a Project

The Interview

When you bootstrap, Recursa asks six key questions:

  1. What is this system for?
  2. What is one cycle of work?
  3. How do you measure success?
  4. What should never happen?
  5. What can be learned?
  6. What personality should it have?

CLI Reference

# Initialize a new project
python tools/recursa.py init ~/my-project

# Validate project structure
python tools/recursa.py validate ~/my-project

# Check project status
python tools/recursa.py status ~/my-project

# Log an action
python tools/recursa.py log ~/my-project -a "edit" -m "Updated SOUL.md"

# Check for migrations
python tools/recursa.py migrate ~/my-project
